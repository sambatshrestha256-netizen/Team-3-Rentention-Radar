## Repository Structure

```
retention-radar/
├── Data_Preparation/       # Stage 2.1 — preprocessing, train/test split, scaling
├── Clustering_Analysis/    # Stage 2.2 — elbow method, K-Means, cluster visualisation
├── Predictive_Modeling/    # Later stage — churn prediction model
└── README.md
```

Each folder contains its own README describing exactly what belongs there and the
naming conventions to follow. See those files before adding content.

## Stage 2 Deliverables

**Data Preparation**
- Preprocessed dataset (missing values handled, categorical variables encoded)
- Train/test split with documentation on size and composition
- Scaling technique documentation with code snippets

**Clustering Analysis**
- Elbow method analysis identifying the optimal number of clusters
- Trained K-Means model + training script/notebook
- Cluster visualisations with labels and interpretation

## Predictive Modeling (Later Stage)

- Churn prediction model training and evaluation
- Feature importance and business insights
